El proyecto en `D:\Iteracion` existe para crear **briefs de iteración de conceptos** de anuncios (Meta/TikTok).

Estructura de carpetas (creada 2026-06-11): `videos/` (input crudo), `analisis-videos/` (guion+tomas), `briefs/` (output final), `documentos/{prompts-research,prompts-creacion,curso,imagenes-referencia}` (frameworks). Hay un `README.md` en la raíz con la convención. Convención: cada video genera `analisis-videos/<slug>--analisis.md` (guion+tomas) y `briefs/<Producto>_<Tema>_Briefs-Iteracion.doc` (documento Word). **ENTREGABLE POR DEFECTO (actualizado 2026-06-18): brief Breakthrough en `.doc` HTML maquetado, 9 conceptos** — ver [[formato-brief-breakthrough]] para el formato/estilo exacto. (CORRECCIÓN de nota vieja: el `.doc` HTML SÍ abre en Word vía COM sin colgarse — el truco es abrirlo en SOLO-LECTURA: `$w.Documents.Open($src,$false,$true)` (3er parámetro ReadOnly=$true bypassa el Protected View que antes lo colgaba). Confirmado en esta sesión: abre, pagina y exporta a PDF.) Alternativa solo si alguna vez se pide `.docx` OOXML nativo (no HTML): script `scripts/md2docx_ooxml.ps1 -Md <in.md> -Docx <out.docx>` (arma el zip con System.IO.Compression; soporta H1-3, bold, italic, code, tablas, listas, blockquote, hr). No hay pandoc/python. **OJO — dos formatos de brief distintos:** (a) el **Breakthrough** (default actual): deconstrucción del ganador + territorio sin explotar + 9 conceptos, cada uno con Script Flow + Guion corrido. (b) el viejo **Perspective-Shift / Creative Strategy Course** (NO usar salvo que el usuario lo pida): Persona → Funnel+Awareness → Tone → Key Messages → Angle → 3 Hooks → Script Flow (Hook→Root Problem→Failed Solution→Root Solution→Producto→Mechanism→Demo→USPs→Results→Social Proof→CTA) → Effects/Music → QC 14 preguntas → Creator Guidelines. Método OCR para subtítulos: extraer frames con ffmpeg (fps=1/2, scale 540), tile 3x3 con etiqueta de segundo (drawtext fontfile C:/Windows/Fonts/arialbd.ttf), copiar a carpeta del proyecto (el Read no ve /tmp de bash) y leerlos con visión.

Flujo de trabajo acordado (2026-06-11):
1. El usuario pasa un video (ad ganador o a iterar).
2. Analizar el video con `ffmpeg`/`ffprobe`: detectar cambios de plano, extraer un frame por toma, y **leer los subtítulos quemados con visión propia (NO tesseract)** para reconstruir el guion + desglose de planos. Tesseract no está instalado; el método nativo (frames → visión) es el preferido. Solo instalar tesseract (`winget install tesseract-ocr`) si un video necesita transcripción literal masiva.
DESCARGA DE VIDEOS (resuelto 2026-06-11): el MCP NO expone el `.mp4`, pero con un **token de Meta con `ads_read`** sí se puede. Truco clave: el `source` del video se obtiene por el edge del ad account (`act_<id>/advideos?fields=id,title,source`) o por el nodo del **advideo asset** (`/<advideo_id>?fields=source`) — NO por el `creative.video_id` (ese es el "reel" publicado, bloqueado). Los uploads se llaman como el ad (`SO6 AD 37.mp4`, `SO3 30_1.mp4`). Script reutilizable: `scripts/fb-fetch.sh <token_file> <act_id> <out_dir> "patron"...` (token se lee de archivo, p.ej. /tmp/fbtok; NUNCA hardcodear). El Bash sí tiene salida a graph.facebook.com y fbcdn. Token de Graph API Explorer dura ~1-2h; recordar revocar.

3. Conectar a Meta vía el MCP oficial `claude.ai Meta_Ads` (ya conectado y verificado). El usuario indica la cuenta + campaña del video para sacar métricas/contexto.
4. Crear los briefs de iteración usando los frameworks de la carpeta (ver [[frameworks-marketing-carpeta]]).
